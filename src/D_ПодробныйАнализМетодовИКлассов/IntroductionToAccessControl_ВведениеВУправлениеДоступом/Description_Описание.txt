Управление доступом (Access Control) в Java - это механизм,
который позволяет ограничивать доступ к членам класса (полям, методам, внутренним классам)
с целью обеспечения инкапсуляции и безопасности программы.
Java предоставляет четыре уровня доступа, которые позволяют классам
и пакетам ограничивать доступ к своим членам:
1. Public (публичный):
Ключевое слово: public
Модификатор public делает член класса доступным для всех классов и пакетов
вне зависимости от их расположения.
Члены с этим модификатором видны из любой точки программы.
2. Private (приватный):
Ключевое слово: private
Модификатор private делает член класса доступным только внутри самого класса.
Доступ к нему могут получать только другие члены этого класса.
Этот уровень доступа обеспечивает наивысшую степень инкапсуляции и безопасности.
3. Protected (защищенный):
Ключевое слово: protected
Модификатор protected делает член класса доступным внутри класса,
внутри подклассов этого класса и внутри того же пакета.
Этот уровень доступа полезен для реализации наследования.
4. Default (по умолчанию) (package-private):
В Java, если не указан никакой модификатор доступа (ни public, ни private, ни protected),
то член класса считается доступным только внутри текущего пакета.
Этот уровень доступа иногда называется "по умолчанию".

Управление доступом важно для обеспечения безопасности, поддержания инкапсуляции и
соблюдения принципов объектно-ориентированного программирования (ООП).
Он позволяет скрывать детали реализации и предоставлять интерфейс,
который другие классы могут использовать без знания внутренней структуры класса.
