Стандартные методы интерфейса (Default Methods) были введены в Java 8
и предоставляют возможность добавлять новые методы в существующие интерфейсы
без нарушения совместимости с классами, которые уже реализовали эти интерфейсы.
Они также называются "методами с реализацией по умолчанию".
Стандартные методы позволяют добавить новые функции в интерфейсы,
не требуя от всех реализующих классов переопределять эти методы.

Основные характеристики стандартных методов:

- Они объявляются с использованием ключевого слова default.
- Они имеют реализацию по умолчанию.
- Они могут быть унаследованы и использованы классами, реализующими интерфейс.
- Если класс реализует несколько интерфейсов и эти интерфейсы имеют одинаковые стандартные
методы, то класс должен предоставить собственную реализацию этого метода.

Пример стандартного метода в интерфейсе:
public interface MyInterface {
    void abstractMethod(); // Абстрактный метод

    default void defaultMethod() {
        System.out.println("Это стандартный метод интерфейса");
    }
}
В этом примере defaultMethod() - это стандартный метод с реализацией по умолчанию.
Классы, реализующие интерфейс MyInterface, могут использовать этот метод без явного
его определения.

Пример использования стандартного метода:
public class MyClass implements MyInterface {
    @Override
    public void abstractMethod() {
        System.out.println("Реализация абстрактного метода");
    }

    public static void main(String[] args) {
        MyClass obj = new MyClass();
        obj.abstractMethod(); // Вызывает реализацию абстрактного метода
        obj.defaultMethod();  // Вызывает стандартный метод
    }
}
Обратите внимание, что стандартные методы интерфейсов могут быть переопределены в классах,
если необходимо предоставить собственную реализацию.
