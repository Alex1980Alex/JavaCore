"Стирание" (type erasure) - это одна из ключевых концепций, связанных с обобщениями (generics) в Java.
Java использует стирание типов для обеспечения обратной совместимости и совместимости с более ранними
версиями языка, так как обобщенные типы были добавлены позднее.

Суть стирания заключается в том, что информация о типах обобщенных объектов удаляется (стирается)
во время компиляции, и в рантайме эти объекты превращаются в обычные объекты. Это делается
для обеспечения безопасности типов в рантайме и сохранения обратной совместимости.

Пример стирания:
public class MyList<T> {
    private T[] array;

    public MyList(int size) {
        array = (T[]) new Object[size]; // Стирание типа
    }

    public T get(int index) {
        return array[index];
    }

    public void set(int index, T value) {
        array[index] = value;
    }
}

В этом примере, при создании массива array, мы создаем массив типа Object, и затем при выполнении
(T[]) происходит явное приведение к типу T. Это приведение типа будет убрано во время компиляции,
и в рантайме array будет обычным массивом Object.

Стратегия стирания позволяет сохранить обратную совместимость с более старыми версиями Java,
но также ведет к ограничениям в использовании обобщенных типов в рантайме и создает некоторые
ограничения в рефлексии и динамическом определении типов.
