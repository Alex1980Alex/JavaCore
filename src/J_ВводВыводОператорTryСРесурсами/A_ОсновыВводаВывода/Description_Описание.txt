Кnассы потоков ввода-вывода байтов из пакета java.io

Потоки ввода-вывода байтов определены в двух иерархиях классов.
На вершине этих иерархий находятся абстрактные классы InputStream и OutputStream.
У каждого из этих абстрактных классов имеется несколько конкретных подклассов,
в которых учитываются отличия разных устройств, в том числе файлов на диске,

Потоки ввода-вывода символов также определены в двух иерархиях классов.
На вершине этих иерархий находятся два абстрактных класса - Reader и Writer.
Эти абстрактные классы управляют потоками символов в Юникоде.
Для каждого из них в Java предусмотрен ряд конкретных подклассов.

Предопределенные потоки данных
Все проrраммы на Java автоматически импортируют пакет java.lang, в котором определен класс System.
Класс System также содержит три предопределенные потоковые переменные: in, out и err.
Они объявлены в System как поля pubic, static и final, т.е. моrут использоваться в любой другой части проrраммы
без привязки к конкретному объекту System.

Поле System.out ссылается на стандартный поток вывода. По умолчаниюэто консоль.
Поле System.in ссылается на стандартный поток ввода, в качестве которого по умолчанию выступает клавиатура.

Systern.in представляет собой объект типа InputStrearn, а Systern.out и Systern.err - объекты типа PrintStream.
Это потоки байтовых данных, хотя обычно они применяются для чтения символов из консоли и записи
символов на консоль.
